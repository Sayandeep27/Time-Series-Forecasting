# 📘 Classical vs STL (LOESS) Decomposition — Complete Notes

---

## 🔹 What to Do After You Get Time Series Data?

### **Step 1: Visualize the Data**

👉 First, always plot your data.

Ask yourself:

* Is there a **trend**? (upward / downward)
* Is there **seasonality**? (daily / weekly / monthly repeating pattern)
* Are seasonal fluctuations:

  * **Constant size** → same height every cycle
  * **Growing/Shrinking** → changes with trend

### ✅ Example:

* Ice cream sales:

  * Every summer increase → **seasonality**
  * Sales increasing every year → **trend**
  * If seasonal peaks grow → multiplicative

---

### **Step 2: Choose Additive vs Multiplicative Model**

| Case                                 | Model              |
| ------------------------------------ | ------------------ |
| Seasonality is constant              | **Additive**       |
| Seasonality grows/shrinks with trend | **Multiplicative** |

#### 📌 Formulas:

```text
Additive Model:
Y(t) = Trend + Seasonality + Residual

Multiplicative Model:
Y(t) = Trend × Seasonality × Residual
```

---

### **Step 3: (Optional) Log Transform**

👉 If data is **multiplicative**, convert to additive:

```text
log(Y) = log(T) + log(S) + log(R)
```

### ✅ Why?

* Because **STL Decomposition only supports Additive Model**

---

### **Step 4: Decompose the Series**

Use either:

* **Classical Decomposition**
* **STL (LOESS) Decomposition**

---

### **Step 5: Analyze Components**

* Study **trend separately**
* Study **seasonality separately**
* Analyze **residuals (noise)**

---

# 🔹 What is Classical Decomposition?

## ✅ Definition:

Classical decomposition is a **traditional time series technique** used to split a time series into three basic components:

* **Trend (T)** → long-term movement
* **Seasonality (S)** → repeating pattern
* **Residual/Noise (R)** → random fluctuations

👉 It can be either **additive or multiplicative**.

---

## ⚙️ How it Works:

* **Trend** → computed using a **moving average**
* **Seasonality** → estimated by averaging seasonal components
* **Residual** → remaining unexplained variations

---

## 📌 Mathematical Formulation (Additive):

```text
Y(t) = Trend(t) + Seasonality(t) + Residual(t)
```

---

## ⚠️ Limitations:

* Assumes **fixed seasonality over time**
* Not suitable for changing patterns

---

# 🔹 What is STL Decomposition (Seasonal-Trend decomposition using LOESS)?

## ✅ Definition:

STL is a **modern and flexible method** for decomposing time series data into:

* Trend
* Seasonality
* Residual

---

## ⚙️ Key Concept: LOESS

**LOESS (Locally Weighted Regression)** is a smoothing technique that:

* Fits **local regression models**
* Produces **smooth and adaptive curves**
* Handles changing patterns effectively

---

## 📌 Mathematical Formulation:

```text
Y(t) = Trend(t) + Seasonality(t) + Residual(t)
```

---

## 💡 Features:

* **Trend** → smooth and dynamic
* **Seasonality** → can change over time
* **Residuals** → better separation of noise

---

## ⚠️ Important Points:

* Supports **only additive model**
* Multiplicative data must be transformed using **log transformation**

---

# 🔹 Classical vs STL — Core Idea

Both methods decompose time series into:

* Trend
* Seasonality
* Residual

👉 The main difference is **flexibility and robustness**

---

# 🔹 When to Use Which?

| Scenario                                 | Best Approach           |
| ---------------------------------------- | ----------------------- |
| Stable Seasonality & Trend               | Classical Decomposition |
| Changing Seasonality Over Time           | STL Decomposition       |
| Missing Data or Irregular Time Intervals | STL Decomposition       |
| Computational Efficiency Needed          | Classical Decomposition |
| Better Forecasting Performance           | STL Decomposition       |

---

# 🔹 Key Differences

| Feature               | Classical Decomposition      | STL (LOESS) Decomposition  |
| --------------------- | ---------------------------- | -------------------------- |
| Method                | Uses moving averages         | Uses LOESS smoothing       |
| Seasonality Handling  | Fixed over time              | Changes over time          |
| Trend Estimation      | Simple moving average        | Flexible smoothing         |
| Residuals             | Less precise                 | More accurate              |
| Handling Missing Data | Sensitive                    | Robust                     |
| Adaptability          | Poor for non-stationary data | Good for changing patterns |
| Computational Cost    | Fast                         | More expensive             |
| Best Use Case         | Stable data                  | Real-world dynamic data    |

---

# 🔹 Simple Intuition

## Classical Decomposition

👉 Assumes pattern is **fixed**

Example:

* Sales increase at a constant rate
* Seasonal peaks remain same every year

---

## STL Decomposition

👉 Assumes pattern can **change over time**

Example:

* Online sales where seasonal peaks increase every year
* Dynamic trend behavior

---

# 🔥 Final Summary (Exam Ready)

* Always **visualize the data first**
* Identify:

  * Trend
  * Seasonality
* Decide:

  * Additive vs Multiplicative
* If multiplicative → apply **log transform**
* Choose method:

  * **Classical** → simple, stable data
  * **STL** → flexible, real-world data
