# 📊 Time Series Model Selection Guide (AR, MA, ARMA, ARIMA, SARIMA, Prophet)

---

## 📌 Overview

This guide helps you quickly decide **which time series model to use** based on your data characteristics.

It covers:

* AR (AutoRegressive)
* MA (Moving Average)
* ARMA
* ARIMA
* SARIMA
* Prophet

---

## 🧠 Quick Decision Cheat Sheet

| Situation                           | Recommended Model |
| ----------------------------------- | ----------------- |
| Only past values matter             | AR                |
| Only noise/shocks matter            | MA                |
| Both effects + stationary           | ARMA              |
| Trend present (non-stationary)      | ARIMA             |
| Trend + seasonality                 | SARIMA            |
| Real-world messy data (fast & easy) | Prophet           |

---

## 🔍 Model-wise Explanation

---

### 🔹 1. AR (AutoRegressive Model)

**Use When:**

* Data depends on **its past values**
* Data is **stationary** (no trend, no seasonality)

**Idea:**

```
X_t = c + φ1*X_{t-1} + φ2*X_{t-2} + ... + ε_t
```

**Key Signals:**

* PACF → Cuts off
* ACF → Tails off

**Example Use Cases:**

* Temperature prediction
* Sensor readings

---

### 🔹 2. MA (Moving Average Model)

**Use When:**

* Data depends on **past errors (noise)**
* Data is **stationary**

**Idea:**

```
X_t = μ + ε_t + θ1*ε_{t-1} + θ2*ε_{t-2}
```

**Key Signals:**

* ACF → Cuts off
* PACF → Tails off

**Example Use Cases:**

* Shock-driven systems
* Error smoothing

---

### 🔹 3. ARMA (AutoRegressive Moving Average)

**Use When:**

* Data depends on **past values + past errors**
* Data is **stationary**

**Idea:**

```
X_t = c + φX_{t-1} + ε_t + θ*ε_{t-1}
```

**Key Signals:**

* ACF → Tails off
* PACF → Tails off

**Example Use Cases:**

* Financial returns
* Stable signals with mixed dependencies

---

### 🔹 4. ARIMA (AutoRegressive Integrated Moving Average)

**Use When:**

* Data is **non-stationary**
* Can be made stationary using **differencing**

**Components:**

* AR (p)
* I (d) → Differencing
* MA (q)

**Idea:**

```
ARIMA(p, d, q)
```

**Example Use Cases:**

* Sales with trend
* Economic indicators

---

### 🔹 5. SARIMA (Seasonal ARIMA)

**Use When:**

* Data has **trend + seasonality**

**Components:**

```
SARIMA(p, d, q)(P, D, Q, m)
```

Where:

* m = seasonal period (e.g., 12 for monthly data)

**Example Use Cases:**

* Monthly sales
* Electricity demand
* Seasonal business metrics

---

### 🔹 6. Prophet (by Meta)

**Use When:**

* You want **fast, practical forecasting**
* Data includes:

  * Trend
  * Multiple seasonality
  * Holidays/events

**Key Features:**

* Handles missing data
* Robust to outliers
* Minimal tuning required

**Example Use Cases:**

* Business forecasting
* Marketing analytics
* Website traffic

---

## 🔄 Model Selection Flow (Step-by-Step)

```
1. Is data stationary?
   ├── YES → Go to Step 2
   └── NO → Use ARIMA (or SARIMA if seasonal)

2. Is there seasonality?
   ├── YES → Use SARIMA
   └── NO → Go to Step 3

3. Dependency type?
   ├── Past values only → AR
   ├── Past errors only → MA
   └── Both → ARMA

4. Want quick business solution?
   └── Use Prophet
```

## ⚖️ Model Comparison Table

| Model   | Handles Trend | Handles Seasonality | Requires Stationarity | Complexity | Use Case             |
| ------- | ------------- | ------------------- | --------------------- | ---------- | -------------------- |
| AR      | ❌             | ❌                   | ✅                     | Low        | Simple patterns      |
| MA      | ❌             | ❌                   | ✅                     | Low        | Noise modeling       |
| ARMA    | ❌             | ❌                   | ✅                     | Medium     | Combined effects     |
| ARIMA   | ✅             | ❌                   | ❌ (after diff)        | Medium     | Trend data           |
| SARIMA  | ✅             | ✅                   | ❌ (after diff)        | High       | Seasonal data        |
| Prophet | ✅             | ✅                   | ❌                     | Low        | Business forecasting |

---

## 🧾 Key Takeaways

* **AR / MA / ARMA** → Use for **stationary data**
* **ARIMA** → Use when data has **trend**
* **SARIMA** → Use when data has **trend + seasonality**
* **Prophet** → Best for **real-world, messy, business data**

---

## 🚀 Bonus Tip

Always follow this workflow:

```
1. Visualize data
2. Check stationarity (ADF test)
3. Apply differencing if needed
4. Check ACF & PACF plots
5. Choose model
6. Validate using metrics (AIC, BIC, RMSE)
```

---

## 📌 Conclusion

Choosing the right time series model depends on:

* Stationarity
* Trend
* Seasonality
* Data complexity

Start simple → Then move to complex models only if needed.
