# 1. 📖 Introduction

In time series analysis, we deal with data over time like:

* Daily stock prices
* Monthly sales
* Temperature over years

To build good models (like ARIMA), the data must behave in a **stable way**.

👉 This stability is called **stationarity**.

---

# 2. 📌 What is Stationarity?

A time series is **stationary** if its statistical properties do NOT change over time.

### In very simple words:

> The pattern of the data stays consistent over time.

---

### Three Main Conditions:

1. **Constant Mean** → Average value stays same
2. **Constant Variance** → Spread of data stays same
3. **Constant Covariance** → Relationship depends only on gap (lag), not time

---

### Example:

✅ Stationary Data:

* Temperature fluctuating around 25°C daily

❌ Non-Stationary Data:

* Sales increasing every year (trend present)

---

# 3. 🔍 Types of Stationarity

## 3.1 Strict Stationarity (Very Strong Condition)

A series is strictly stationary if:

> The COMPLETE distribution of data does not change over time

### Easy Understanding:

Imagine taking data from:

* Year 2000
* Year 2020

If both have EXACT same distribution → strictly stationary

### Includes:

* Mean
* Variance
* Skewness
* Kurtosis
* Entire shape of distribution

---

## 3.2 Weak Stationarity (What we actually use)

This is a relaxed version.

A series is weakly stationary if:

* Mean is constant
* Variance is constant
* Covariance depends only on lag

👉 This is enough for most models like ARIMA

---

# 4. ❗ Why Stationarity is Important

### Without stationarity:

* Models become unreliable
* Predictions become wrong
* Patterns keep changing

### With stationarity:

* Easier to model
* Better forecasting
* Stable relationships

👉 That’s why we ALWAYS check stationarity before modeling

---

# 5. ❗ Can We Test Strict Stationarity?

### ❌ No (Not Practical)

Why?

* Need to compare full distributions at all time points
* Requires infinite data
* Impossible in real-world

### ✔ So what do we do?

We focus only on:

👉 **Weak Stationarity (testable)**

---

# 6. 🧪 Tests for Weak Stationarity (Detailed)

---

## 6.1 📈 Visual Inspection (First Step)

### Plot the data and check:

| Pattern               | Meaning               |
| --------------------- | --------------------- |
| Upward/Downward trend | ❌ Not stationary      |
| Increasing spread     | ❌ Not stationary      |
| Constant fluctuation  | ✅ Possibly stationary |

👉 Always start with plotting the graph

---

## 6.2 📊 Rolling Statistics

### Idea:

Check if mean and variance change over time

### How it works:

* Take a window (e.g., 12 months)
* Compute mean & std repeatedly

### If:

* Rolling mean is flat → good
* Rolling std is flat → good

👉 Then data is likely stationary

---

### Python Code:

```python
rolling_mean = data.rolling(window=12).mean()
rolling_std = data.rolling(window=12).std()
```

---

## 6.3 📉 Augmented Dickey-Fuller (ADF) Test

### Core Idea:

Checks if the series has a **unit root** (which means non-stationary)

---

### Hypothesis:

* H₀: Data is **non-stationary** ❌
* H₁: Data is **stationary** ✅

---

### Decision Rule:

| p-value | Meaning                        |
| ------- | ------------------------------ |
| < 0.05  | Reject H₀ → Stationary         |
| ≥ 0.05  | Cannot reject → Non-stationary |

---

### Intuition (Very Important):

ADF checks:

👉 "Does past value strongly influence future?"

If YES → trend exists → non-stationary

---

### Python Code:

```python
from statsmodels.tsa.stattools import adfuller

result = adfuller(data)

print("ADF Statistic:", result[0])
print("p-value:", result[1])
```

---

## 6.4 📉 KPSS Test

### Core Idea:

Checks if data is already stationary

---

### Hypothesis:

* H₀: Data is **stationary** ✅
* H₁: Data is **non-stationary** ❌

---

### Decision Rule:

| p-value | Meaning                    |
| ------- | -------------------------- |
| < 0.05  | Reject H₀ → Non-stationary |
| ≥ 0.05  | Stationary                 |

---

### Key Insight:

👉 KPSS is opposite of ADF

---

### Python Code:

```python
from statsmodels.tsa.stattools import kpss

result = kpss(data)

print("KPSS Statistic:", result[0])
print("p-value:", result[1])
```

---

## 6.5 📉 Phillips-Perron (PP) Test

### Core Idea:

* Similar to ADF
* More robust when:

  * Autocorrelation exists
  * Variance is not constant

---

### Hypothesis:

* H₀: Non-stationary
* H₁: Stationary

---

# 7. ⚖️ ADF vs KPSS (Deep Understanding)

### Why use both?

Because they test **opposite things**

---

| Test | Null Hypothesis | If p < 0.05    |
| ---- | --------------- | -------------- |
| ADF  | Non-stationary  | Stationary     |
| KPSS | Stationary      | Non-stationary |

---

### Combined Interpretation:

| ADF            | KPSS           | Final Meaning             |
| -------------- | -------------- | ------------------------- |
| Stationary     | Stationary     | Strong evidence           |
| Non-stationary | Non-stationary | Definitely non-stationary |
| Conflict       | Conflict       | Need deeper analysis      |

---

# 8. 🧭 Step-by-Step Workflow

### Step 1: Plot the data

### Step 2: Check trend & variance

### Step 3: Apply ADF test

### Step 4: Apply KPSS test

### Step 5: Compare results

### Step 6: If not stationary → transform data

---

# 9. 🧾 Summary

### Strict Stationarity:

* Theoretical
* Cannot be tested

---

### Weak Stationarity:

* Practical
* Tested using:

  * ADF
  * KPSS
  * PP Test
  * Rolling stats
  * Visual plots

---

## 🚀 Final Takeaway

> In real-world time series problems, always focus on **weak stationarity** and verify it using ADF + KPSS together.
